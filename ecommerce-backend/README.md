# E-Commerce REST API Backend

A production-ready, scalable backend architecture for an e-commerce platform built with Node.js, Express, and MongoDB.

## Features
- **User System:** Secure JWT Hybrid Authentication (Access & HttpOnly Refresh Tokens), Session tracking, Profile management.
- **Roles:** User & Admin role-based access control.
- **Products & Categories:** Full CRUD with advanced filtering, sorting, pagination, and text search.
- **Shopping Cart:** Persistent cart system with automatic total calculation.
- **Orders & Checkout:** Order processing, status tracking.
- **Security:** Rate Limiting, Helmet HTTP Headers, CORS, Zod validation, Data sanitization.
- **DevOps:** Dockerized, environment-based configuration, Winston logging.
- **Documentation:** Built-in Swagger API Docs.

## Prerequisites
- Node.js (v18 or higher)
- Docker & Docker Compose (optional, for easy MongoDB setup)
- MongoDB (if not using Docker)

## Getting Started

### 1. Clone & Install
\`\`\`bash
npm install
\`\`\`

### 2. Configure Environment variables
Ensure you have a `.env` file in the root directory. 
\`\`\`env
PORT=5000
MONGO_URI=mongodb://localhost:27017/ecommerce
JWT_SECRET=supersecretjwtkey
JWT_EXPIRES_IN=1h
JWT_REFRESH_SECRET=supersecretrefreshkey
JWT_REFRESH_EXPIRES_IN=7d
NODE_ENV=development
\`\`\`
*(For Docker, the `MONGO_URI` is already configured in docker-compose.yml as `mongodb://mongo:27017/ecommerce`)*

### 3. Run the Application

**Option A: Using Docker (Recommended)**
This will spin up both the Node app and a MongoDB instance.
\`\`\`bash
docker-compose up --build
\`\`\`

**Option B: Local Development**
Make sure MongoDB is running locally on port 27017.
\`\`\`bash
npm run dev
\`\`\`

## API Documentation
Once the server is running, the API documentation is available via Swagger UI:
👉 **[http://localhost:5000/api-docs](http://localhost:5000/api-docs)**

## Folder Structure
- `src/config`: Configuration files (DB connection)
- `src/controllers`: Route handlers and business logic
- `src/docs`: Swagger documentation definition
- `src/middlewares`: Global and route-specific middlewares (auth, errors)
- `src/models`: Mongoose database schemas
- `src/routes`: Express route definitions
- `src/utils`: Helper utilities (custom error, catchAsync, API Features, User auth)
- `logs/`: Application logs (generated by Winston)

## API Endpoints (Quick Reference)
- `POST /api/v1/auth/register` - Create an account
- `POST /api/v1/auth/login` - Login to account
- `GET /api/v1/products` - Get all products (supports `?sort=-price&limit=10&search=keyword`)
- `GET /api/v1/cart` - View cart
- `POST /api/v1/orders` - Place order
